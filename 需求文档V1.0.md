# 智能站会-需求文档 V1.0

目标：生成可行可落地可用的一套系统或软件，不是一个玩具或者demo。

如果项目功能偏多，规模比较大，可以采取敏捷开发的方式进行最小可执行单元进行迭代。



## 1、系统里面的角色组成

谁使用该套系统。里面会涉及到哪些角色，不同的角色之间应该要有哪些权限。（明确）

一共有3种角色、分别是 团队负责人(Scrum Master)、团队成员、系统管理员

角色对应的权限有哪些呢？（数据维护的主体是谁？）

### 1.1、团队负责人（Scrum Master）

A、账号由管理员创建或邀请加入团队

B、登录功能、退出功能

C、个人管理（补充个人信息、设置偏好）

D、创建和管理站会（设置站会周期、时间、参与成员、站会模板）

E、查看团队所有成员的站会纪要（按日期、成员筛选）

F、审核和管理自动提取的行动项（确认、修改、指派负责人和截止时间）

G、任务追踪面板（查看所有行动项状态，标记完成/阻塞/进行中）

H、团队效能数据看板（站会参与率、行动项完成率、团队速率、燃尽图）

I、团队成员管理（邀请、移除、角色分配）

J、站会模板管理（自定义站会问题模板）

K、查看异常预警（连续未提交站会、行动项逾期等）

### 1.2、团队成员

A、账号由管理员/团队负责人创建或邀请注册

B、登录功能、退出功能

C、个人管理（补充个人信息）

D、参加每日站会（文字输入 / 语音输入 回答站会三个问题：昨天做了什么、今天计划做什么、有什么阻塞）

E、查看自己的历史站会记录

F、查看和更新分配给自己的行动项（标记状态、添加备注）

G、查看团队的共享看板（了解他人进展）

H、接收通知（站会提醒、行动项到期提醒、@提及）

I、异步站会（无法实时参加时，在截止时间前异步提交站会内容）

### 1.3、系统管理员

A、超级管理员账号，由系统直接提供，不可注册

B、登录功能、退出功能

C、团队管理（创建团队、解散团队、修改团队信息）

D、用户管理（管理所有用户账号、重置密码、冻结/解冻账号）

E、系统配置管理（全局参数设置、站会时间窗口配置、预警阈值配置）

F、数据总览（全平台数据统计与分析）

G、操作日志查看（审计日志、异常日志查询）

H、集成配置管理（飞书/企业微信/钉钉等第三方应用对接配置）



## 2、系统核心功能模块

### 2.1、智能站会纪要模块（核心）

**2.1.1 站会创建与配置**
- 团队负责人创建站会，设置周期（每日/每周）、时间窗口、参与成员
- 自定义站会三个标准问题的题干（默认：昨天做了什么 / 今天计划做什么 / 有什么阻塞）
- 支持按团队设置不同的站会模板

**2.1.2 站会提交**
- 支持文字输入方式填写站会内容
- 支持语音输入，系统自动语音转文字
- 支持移动端（小程序/H5）和PC Web端提交
- 站会提交窗口内系统自动提醒，超时未提交标记为缺席

**2.1.3 AI智能识别与提取**
- 自动识别站会内容中的"昨日完成 / 今日计划 / 阻塞项"三部分，进行结构化拆分
- 自动提取行动项（Action Items），识别"谁、做什么、什么时候完成"
- 自动关联上一次站会中提到的行动项，判断是否完成

**2.1.4 站会纪要生成**
- 自动生成结构化的每日站会纪要（包含：参与人员、每人三问回答、行动项清单、阻塞项清单）
- 支持一键分发到飞书/企业微信/钉钉群
- 支持按日期、成员多维度检索历史纪要

### 2.2、任务追踪闭环模块（核心）

**2.2.1 行动项自动提取**
- 从站会纪要中AI自动提取行动项，生成结构化任务卡片
- 智能识别负责人（通过上下文和@提及）、截止时间（自然语言解析）
- 支持人工确认、修改、补充行动项信息

**2.2.2 任务状态管理**
- 行动项状态流转：待处理 → 进行中 → 已完成 / 已阻塞
- 设置截止时间，逾期自动预警
- 支持添加任务备注、附件、关联站会记录

**2.2.3 任务回顾与追溯**
- 下次站会自动回顾上次未完成的行动项
- 生成个人/团队任务完成统计报表
- 行动项全生命周期追溯（谁创建、何时分配、何时完成）

### 2.3、团队效能分析模块

**2.3.1 站会数据统计**
- 站会参与率统计（出勤率、迟到率、缺席率）
- 人均行动项数量统计
- 行动项完成率、平均完成周期统计
- 阻塞项类型分布统计（技术受阻 / 资源不足 / 外部依赖 / 沟通协调等）

**2.3.2 团队健康度分析**
- 燃尽图：展示迭代周期内剩余工作量的变化趋势
- 团队速率图：每个迭代完成的工作量趋势
- 阻塞项趋势：反映团队协作顺畅度
- 成员贡献度：均衡性分析，识别过载和闲置

**2.3.3 效能报告**
- 自动生成周报/月报/迭代报告
- 团队效能对比（多团队场景）
- 数据导出（Excel/PDF）

### 2.4、异步协同模块

**2.4.1 异步站会**
- 无法实时参加的成员可在截止时间前异步提交站会
- 已提交内容对所有成员可见，支持评论互动
- 异步提交同样走AI识别和行动项提取流程

**2.4.2 多平台同步**
- PC Web端全功能管理后台
- 移动端（微信小程序/H5）轻量级站会提交与查看
- 集成飞书/企业微信/钉钉消息推送（站会提醒、纪要分发、行动项通知）

### 2.5、看板与可视化模块

**2.5.1 任务看板**
- 看板视图展示所有行动项（待处理、进行中、已完成、已阻塞）
- 支持拖拽改变任务状态
- 支持按负责人、优先级、截止时间筛选

**2.5.2 项目进度视图**
- 迭代进度总览
- 燃尽图实时更新
- 成员负载视图（判断谁任务过多、谁有空闲）



## 3、系统的部署平台

### 3.1、PC Web端（管理后台 + 全功能）

技术栈：SpringBoot + MybatisPlus（后端），Vue3（前端）

面向角色：团队负责人、系统管理员

主要功能：团队管理、站会配置、数据看板、效能分析、任务管理

### 3.2、移动端（小程序/H5）

技术栈：微信小程序原生 / uniapp（跨端）

面向角色：团队成员

主要功能：站会提交（文字+语音）、查看纪要、任务状态更新、消息通知

### 3.3、第三方平台集成

- 飞书开放平台接入
- 企业微信应用接入
- 钉钉应用接入

主要能力：消息推送、纪要分发、行动项提醒



## 4、智能化能力

### 4.1、语音转文字
- 成员可通过语音输入站会内容，系统自动转为文字
- 支持中文普通话（V1.0），后续可扩展方言和英文

### 4.2、NLP智能提取
- 从非结构化站会文本中自动识别"完成/计划/阻塞"三段式内容
- 自动提取行动项（命名实体识别 + 意图分类）
- 支持自然语言时间解析（"明天""下周""本周五" → 具体日期）

### 4.3、智能提醒与预警
- 站会时间窗口开启时自动推送提醒
- 行动项即将到期/已逾期自动预警
- 连续缺席站会预警通知团队负责人
- 阻塞项超过N天未解决自动升级告警



## 5、技术约束与非功能需求

### 5.1、数据安全
- 用户密码加密存储
- 站会内容数据传输加密（HTTPS）
- 敏感数据脱敏处理
- 数据库定期自动备份

### 5.2、性能要求
- 页面加载时间 < 2秒
- 语音转文字响应时间 < 5秒
- 支持至少100人同时在线提交站会
- 纪要看板数据实时刷新延迟 < 3秒

### 5.3、可用性
- 系统可用性 ≥ 99.5%
- 支持主流浏览器（Chrome、Edge、Firefox）
- 移动端兼容iOS 14+、Android 10+

### 5.4、可扩展性
- 模块化设计，各功能模块松耦合
- 预留第三方工具集成接口（Jira、TAPD、飞书文档等）
- 支持多团队、多项目隔离
